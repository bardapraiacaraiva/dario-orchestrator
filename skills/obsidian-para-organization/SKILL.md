# OBSIDIAN-PARA-ORGANIZATION

## Princípio (Tiago Forte)
**Organize by actionability, not by topic.** Move-se entre os 4 buckets conforme estado.

## Os 4 buckets
- **Projects:** outcome específico + deadline (active commitments)
- **Areas:** standards a manter no tempo (não terminam)
- **Resources:** tópicos de interesse (referência)
- **Archive:** projects/areas inactive

## Quando usar
- Vault com 1000+ notes sem estrutura
- Notion/Drive folder mess
- Migrate from topic-based → PARA
- Cross-app PARA (Obsidian + Drive + Notion same)

## Templates
1. PARA root folder structure
2. Project naming convention (`P - 2026-Q2 - Atrium Rebuild`)
3. Project completion workflow (project → archive)
4. Weekly review (sweep Inbox + reorganize)
5. Quarterly PARA audit (active projects only?)

## Princípios
- **Recency over completeness:** active projects on top
- **Inbox = temp:** triage para PARA semanalmente
- **Archive over delete:** memory tem custo zero
- **Same structure across tools:** Obsidian == Drive == Notion folders
