# ORION-PRODUCT-DISCOVERY

## Quando usar
- Bootstrap de programa de discovery (greenfield)
- Auditoria de processo existente (estão a testar assumptions ou só shipping?)
- Setup de cadência semanal de entrevistas
- Build opportunity solution tree
- Assumption mapping pré-build

## Princípios (Teresa Torres)
- **Weekly customer interviews:** mínimo 3 por semana
- **Opportunity solution tree:** outcome → opportunities → solutions → experiments
- **Assumption mapping:** desejabilidade / viabilidade / factibilidade / usabilidade
- **Generative + evaluative:** descobrir problemas E validar soluções

## Templates
1. Interview script (semi-structured, ~30 min)
2. Opportunity solution tree (Miro/FigJam)
3. Assumption test plan (riskiest assumption first)
4. Weekly discovery review cadence
5. Insight repository structure (Dovetail/Notion)
