## Workflow

### Step 1: Value Equation Foundation (Hormozi)

The Grand Slam Offer formula:

```
            Dream Outcome x Perceived Likelihood of Achievement
Value = ─────────────────────────────────────────────────────────
              Time Delay x Effort & Sacrifice
```

**Goal**: Maximize the numerator, minimize the denominator.

| Component | Current State | Optimized State | How |
|-----------|--------------|-----------------|-----|
| **Dream Outcome** | What the customer thinks they'll get | Make it vivid, specific, measurable | [specific improvement] |
| **Perceived Likelihood** | How likely they believe it'll work | Increase with proof, guarantees, specificity | [specific improvement] |
| **Time Delay** | How long until they see results | Shorten with quick wins, fast onboarding | [specific improvement] |
| **Effort & Sacrifice** | How much work they must do | Reduce with done-for-you, automation, simplicity | [specific improvement] |

### Step 2: Dream Outcome Definition

Write the dream outcome in the customer's own words:
- "I want to [SPECIFIC RESULT] so that I can [DEEPER DESIRE]"
- Make it: Measurable, Time-bound, Vivid, Emotionally charged

**Before/After Transformation:**

| Dimension | BEFORE (pain) | AFTER (dream) |
|-----------|--------------|---------------|
| **Have** (tangible) | [what they have now] | [what they'll have] |
| **Feel** (emotional) | [how they feel now] | [how they'll feel] |
| **Status** (social) | [how others see them] | [how others will see them] |
| **Daily life** (practical) | [typical day now] | [typical day after] |

### Step 3: Delivery Cube (Hormozi)

Design the delivery mechanism across 4 dimensions:

| Dimension | Options | Your Choice | Rationale |
|-----------|---------|-------------|-----------|
| **1:1 vs 1:Many** | Personal / Group / Self-paced | | [why] |
| **DIY vs DFY** | Do It Yourself / Done With You / Done For You | | [why] |
| **Virtual vs In-Person** | Online / Hybrid / Physical | | [why] |
| **Speed** | Immediate / Drip / Cohort | | [why] |

**Profit Maximizer Tip**: Offer DIY at lowest price, DWY at mid, DFY at highest.

### Step 4: Offer Stack Construction

List EVERY component the customer receives:

| # | Component | Type | Standalone Value | Delivery |
|---|-----------|------|-----------------|----------|
| 1 | **Core Offer** | Main product/service | $X | [how delivered] |
| 2 | **Bonus 1** | [name] | $X | [how delivered] |
| 3 | **Bonus 2** | [name] | $X | [how delivered] |
| 4 | **Bonus 3** | [name] | $X | [how delivered] |
| 5 | **Bonus 4** | [name] | $X | [how delivered] |
| 6 | **Fast-Action Bonus** | [name] | $X | [first X buyers] |
| | **Total Value** | | **$X** | |
| | **Your Price** | | **$X** | |
| | **Value-to-Price Ratio** | | **X:1** | Aim for 10:1+ |

**Bonus Design Rules (Hormozi):**
1. Each bonus should address a SPECIFIC objection
2. Each bonus should be low cost to deliver, high perceived value
3. Name each bonus with a benefit-driven title
4. Assign a credible standalone dollar value
5. At least one bonus should address "but what if I fail?" (templates, checklists)
6. At least one bonus should address "but I don't have time" (shortcuts, done-for-you elements)

### Step 5: Guarantee Design

Choose and craft the guarantee:

| Guarantee Type | Risk Level (for you) | Conversion Impact | Best For |
|---------------|---------------------|-------------------|----------|
| **Unconditional** | "Full refund, no questions asked, 30 days" | Very High | High-margin products |
| **Conditional** | "Do X, Y, Z and if no result, full refund" | High | Programs, courses |
| **Anti-guarantee** | "All sales final — this is for serious people only" | Medium | Ultra-premium, exclusive |
| **Better-than-money-back** | "Refund + keep bonuses + $X compensation" | Very High | High confidence offers |
| **Performance** | "We'll work free until you get [result]" | Very High | Service businesses |
| **Revenue-share** | "You only pay when you make money" | Maximum | High-ticket consulting |

**Guarantee Statement:**
"[GUARANTEE TYPE]: [SPECIFIC PROMISE]. If [CONDITION], we will [ACTION]. You have [TIMEFRAME] to decide."

### Step 6: Pricing Tiers

Design 3 tiers (Rule of 3):

| Element | Tier 1: [Name] | Tier 2: [Name] | Tier 3: [Name] |
|---------|----------------|----------------|----------------|
| **Price** | $X | $X | $X |
| **Core Offer** | Basic version | Full version | Full + premium |
| **Bonus 1** | No | Yes | Yes |
| **Bonus 2** | No | No | Yes |
| **Bonus 3** | No | No | Yes |
| **Support** | Community | Group calls | 1:1 access |
| **Guarantee** | 14-day | 30-day | 60-day |
| **Target buyer** | Budget-conscious | Best value (anchor) | VIP/serious |
| **Margin** | Low | Highest | High |

**Pricing Psychology:**
- Tier 2 should be the "obvious choice" (highest margin, best value framing)
- Tier 3 exists to make Tier 2 look reasonable (price anchoring)
- Tier 1 exists as a fallback / lead generation entry

### Step 7: Urgency & Scarcity

**Legitimate scarcity mechanisms:**

| Mechanism | Type | Example |
|-----------|------|---------|
| **Cohort-based** | Real scarcity | "Next cohort starts [DATE], 20 spots" |
| **Founding pricing** | Time-bound | "Price goes up to $X on [DATE]" |
| **Capacity limit** | Real scarcity | "I can only take 5 clients/month" |
| **Bonus expiry** | Time-bound | "Fast-action bonus disappears in 48h" |
| **Seasonal** | Calendar-bound | "Q1 pricing ends March 31" |

**Never fake scarcity.** It destroys trust and kills long-term business.

### Step 8: Naming the Offer

**Offer Name Formula:**
[Adjective] + [Mechanism] + [Promise] + [Time/Speed]

Examples:
- "The 90-Day Revenue Accelerator"
- "The Profit-First Launch System"
- "The Magnetic Client Blueprint"

Test 5 names and choose the one that communicates VALUE most clearly.

### Step 9: Offer Litmus Tests

Before launching, the offer must pass these tests:

| Test | Question | Pass? |
|------|----------|-------|
| **No-brainer** | Would YOU buy this at this price? | Y/N |
| **10x value** | Is perceived value at least 10x the price? | Y/N |
| **One-sentence** | Can you explain it in one sentence? | Y/N |
| **Objection coverage** | Does the stack address top 3 objections? | Y/N |
| **Risk reversal** | Does the guarantee remove ALL risk from buyer? | Y/N |
| **Differentiation** | Is this clearly different from competitors? | Y/N |
| **Deliverability** | Can you actually deliver everything promised? | Y/N |
| **Profitability** | Is gross margin above 60%? | Y/N |

## Red Flags

Stop and warn the user if:
- Value-to-price ratio is below 3:1 (too expensive for what's included)
- No guarantee offered (major conversion killer for new businesses)
- User cannot actually deliver what the offer promises (ethics violation)
- Pricing is based on competitor matching instead of value creation
- Offer is too complicated to explain in one sentence
- Bonuses are random filler with no connection to customer objections
- Using fake scarcity or countdown timers that reset (trust destruction)
- Gross margin on the offer is below 40% (unsustainable)
- The offer name is generic and forgettable
