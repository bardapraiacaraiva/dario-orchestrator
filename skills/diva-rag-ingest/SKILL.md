# DIVA Skill — RAG Ingest (Architecture/Design/Construction)

Takes any piece of architecture, design, or construction knowledge and adds it to the DIVA RAG collection with domain-specific metadata. Same engine as dario-rag-ingest (localhost:8420) but uses the `diva` collection and architecture-specific tagging taxonomy.

## When to activate

- User says "guarda isto na RAG DIVA" / "index this for DIVA"
- User shares a PDF of project specifications, building regulations, or supplier catalogs
- User pastes construction details, material specs, or design references
- After completing a DIVA deliverable that should be searchable (energy cert, smart home spec, etc.)
- When ingesting Portuguese building regulations (RGEU, RJUE, REH, RECS, etc.)
- When adding supplier product data (Revigrés, Margres, Vicaima, Saint-Gobain, etc.)
- When indexing project documentation (cadernos de encargos, memorias descritivas)

Do NOT use for:
- Marketing, SEO, SaaS content — use `dario-rag-ingest` instead
- Client-confidential drawings or BIM files (not suitable for text RAG)
- Binary CAD files (.dwg, .rvt, .skp) — extract text notes first
- Content shorter than 50 chars (no retrieval value)

## Content domains and auto-tagging

When ingesting, analyze the content and auto-assign tags from this taxonomy:

| Domain | Tags | Content examples |
|---|---|---|
| Architecture | `architecture`, `design`, `spatial` | Floor plans descriptions, building programs, design concepts |
| Interiors | `interiors`, `finishes`, `furniture`, `decoration` | Material palettes, furniture specs, finish schedules |
| Construction | `construction`, `methodology`, `site` | Construction methods, phasing plans, site logistics |
| Structural | `structural`, `foundations`, `steel`, `concrete`, `timber` | Structural calculations notes, foundation specs |
| MEP | `mep`, `hvac`, `plumbing`, `electrical` | HVAC specs, electrical layouts, plumbing schemes |
| Energy | `energy`, `sce`, `nzeb`, `insulation`, `renewables` | Energy certs, U-value calcs, solar specs |
| Materials | `materials`, `stone`, `ceramic`, `wood`, `metal`, `glass` | Product datasheets, supplier catalogs |
| Regulations | `regulations`, `rgeu`, `rjue`, `reh`, `recs`, `pdr` | Building codes, municipal regulations, licensing |
| Landscape | `landscape`, `garden`, `irrigation`, `hardscape` | Planting plans, irrigation specs, paving |
| Domotica | `domotica`, `knx`, `zigbee`, `automation` | Smart home specs, protocol guides |
| Acoustics | `acoustics`, `insulation`, `reverberation` | Acoustic reports, sound insulation specs |
| Sustainability | `sustainability`, `leed`, `breeam`, `lifecycle` | Certification guides, LCA data |
| Budget | `budget`, `costs`, `quantities`, `pricing` | Cost estimates, BOQs, supplier quotes |
| Suppliers | `supplier`, `portugal`, `catalog` | Product catalogs, supplier contacts |

Always include:
- 1 **domain tag** from above
- 1-2 **specific topic tags**
- 1 **project tag** if project-specific
- 1 **content-type tag:** `spec`, `regulation`, `catalog`, `report`, `guide`, `datasheet`, `notes`

## Workflow

### 1. Identify source type

- **File path** — `.md`, `.pdf`, `.txt`, `.docx`, `.xlsx` (project specs, BOQs, regulations)
- **URL** — supplier websites, regulation portals, technical guides
- **Raw text** — user pasted content, meeting notes, site observations
- **DIVA output** — generated by another DIVA skill (energy cert, smart home spec)

### 2. Derive metadata

- **Name:** `diva/<domain>/<slug>` format
- **Collection:** always `diva` (separate from DARIO's `dario` collection)
- **Source type:** `markdown` / `pdf` / `web` / `note` / `code` / `docx` / `excel`
- **Tags:** auto-derived from content analysis (see taxonomy above)

### 3. Check RAG engine health

```
mcp__dario-rag__kb_health()
```

If not healthy, instruct user:
```bash
cd /c/dario-rag/engine && .venv/Scripts/python.exe main.py &
```

### 4. Call the ingest endpoint

#### For text / notes / DIVA outputs:
```
mcp__dario-rag__ingest_text(
  content: "<full markdown content>",
  name: "diva/<domain>/<slug>",
  source_type: "markdown",
  collection: "diva",
  tags: ["architecture", "<domain>", "<topic>", "<project>"]
)
```

#### For URLs (supplier catalogs, regulation portals, technical guides):
```
mcp__dario-rag__ingest_url(
  url: "https://...",
  collection: "diva",
  tags: ["supplier", "<brand>", "<material-type>"],
  name: "diva/suppliers/<brand>-<product>"
)
```

#### For files (PDFs of cadernos de encargos, project specs):
```bash
curl -X POST http://localhost:8420/ingest/file \
  -F "file=@path/to/caderno-encargos.pdf" \
  -F "collection=diva" \
  -F "tags=construction,spec,<project>"
```

### 5. Verify ingestion

- Check response: `status: "ok"` and `chunks: N > 0`
- If `status: "duplicate"`, content unchanged — OK, inform user
- If error: check engine status, file readability, content length

### 6. Confirm to user

Format:
```
Ingerido na colecao DIVA como "diva/<domain>/<slug>" (N chunks)
Tags: [tag1, tag2, tag3]
Pesquisavel com: mcp__dario-rag__search_kb(query: "...", collection: "diva")
```

## Red flags — don't ingest

- Secrets, credentials, API keys
- Client confidential drawings or full project files without permission
- Unparseable content (scanned PDFs without OCR — suggest OCR first)
- Content shorter than 50 chars (no retrieval value)
- Binary CAD/BIM files — extract text metadata first
- Duplicate content already in the collection (check with `search_kb` first)
- Commercially sensitive supplier pricing that shouldn't be persisted

## Interactions

- Called by `diva-obsidian-save` after saving a file (immediate searchability)
- Called by `diva-projeto` when a project needs its docs indexed
- Called manually when user shares architecture/construction knowledge
- Uses same engine as `dario-rag-ingest` but different collection (`diva` vs `dario`)
- Can cross-reference: `search_kb(query: "...", collection: "diva")` for architecture, `search_kb(query: "...", collection: "dario")` for business
