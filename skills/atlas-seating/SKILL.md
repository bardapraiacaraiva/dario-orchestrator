# atlas-seating — Seating Plans & Table Management

## Description
Comprehensive seating plan design and table management for events of every type — from intimate dinners to 1000+ galas. Covers seating strategies, table configurations, consideration matrices, floor plan layout, VIP management, corporate and social protocols, digital tools, last-minute changes, and output deliverables. Produces visual floor plans and alphabetical/table-by-table guest lists ready for production.

## Trigger Phrases
**EN:** "seating plan", "table assignment", "floor plan", "table layout", "seating chart", "place cards", "escort cards", "table configuration", "head table", "VIP seating"
**PT:** "plano de mesas", "disposicao de mesas", "planta da sala", "mapa de lugares", "cartoes de lugar", "marcadores de mesa", "mesa de honra", "configuracao de sala"

## When to Activate
- Creating or revising a seating plan for any event
- Choosing table configurations and room layout
- Managing VIP and protocol seating requirements
- Designing floor plans with service routes and emergency access
- Handling last-minute seating changes (no-shows, surprise guests)
- Preparing escort cards, place cards, and printed seating charts
- Optimizing networking opportunities through strategic placement
- Resolving conflicts (competitor separation, family dynamics)

## Workflow

### Step 1 — Choose Seating Strategy

| Strategy | Best For | Formality | Complexity |
|----------|----------|-----------|------------|
| **Assigned Seating** (lugar marcado) | Galas, jantares formais, casamentos | Alta | Alta |
| **Table Assignment** (mesa atribuida, lugar livre) | Conferencias, eventos corporativos | Media | Media |
| **Open Seating** (livre) | Cocktails, eventos casuais, workshops | Baixa | Baixa |
| **Mixed** (VIP assigned + geral livre) | Grandes eventos com hierarquia | Media-Alta | Media |
| **Lounge Zones** (zonas tematicas) | Networking events, product launches | Baixa | Baixa |

### Step 2 — Select Table Configurations

| Tipo | Pessoas | Dimensoes | Espaco Total (c/ cadeiras) | Ideal Para |
|------|---------|-----------|---------------------------|------------|
| **Redonda 150cm** | 8 | O 150cm | 3.5m x 3.5m | Jantares formais, galas |
| **Redonda 180cm** | 10 | O 180cm | 4.0m x 4.0m | Galas grandes, casamentos |
| **Rectangular** | 6-12 | 180-360cm x 90cm | Variavel | Banquetes, conferencias |
| **Imperial (longa)** | 20-40 | 600-1200cm x 90cm | Variavel | Family-style, medieval |
| **High-top / Cocktail** | 4 | O 60-80cm, h=110cm | 2.0m x 2.0m | Cocktails, networking |
| **Lounge** | 4-6 | Sofas 200x80cm | 4.0m x 3.0m | VIP areas, casual zones |
| **Cabaret** (meia-lua) | 5-6 | O 180cm (metade) | 4.0m x 2.5m | Conferencias com palco |
| **Classroom** | 2-3/mesa | 180x60cm | 2.5m x 1.5m | Workshops, formacoes |
| **U-shape** | 15-30 | Variavel | Sala inteira | Board meetings, reunioes |
| **Boardroom** | 12-20 | 400-600x150cm | Sala inteira | Reunioes executivas |

### Step 3 — Build the Consideration Matrix

Antes de atribuir lugares, avaliar cada convidado nestas dimensoes:

1. **Hierarquia/Protocolo:** Cargo institucional, ordem de precedencia
2. **Relacao com anfitriao:** Cliente, parceiro, fornecedor, amigo, familia
3. **Idioma:** Agrupar por lingua comum para facilitar conversa
4. **Setor/Industria:** Networking value — juntar complementares, separar concorrentes
5. **Conflitos conhecidos:** Separar rivais, ex-parceiros, disputas comerciais (min. 2 mesas distancia)
6. **Requisitos alimentares:** Facilitar servico agrupando dietas especiais
7. **Acessibilidade:** Lugar acessivel para cadeira de rodas (ponta de mesa, corredor largo)
8. **Networking goals:** Quem queremos que se conheca? Colocar na mesma mesa
9. **Acompanhantes:** Casais juntos (a menos que seja networking puro), familias agrupadas
10. **Repeticao:** Se e evento recorrente, variar vizinhos de mesa para ampliar rede

### Step 4 — Design Floor Plan Layout

**Regras de espacamento:**
- Minimo 120cm entre costas de cadeiras de mesas adjacentes (para servico de catering)
- Minimo 150cm para acesso de cadeira de rodas
- Corredores de servico: minimo 90cm
- Distancia palco-primeira fila: minimo 200cm
- Pista de danca: 0.5-1.0m2 por convidado dancante (tipicamente 30-40% dos convidados)
- Saidas de emergencia: nunca bloquear, sinalizacao visivel de todas as mesas

**Zonas do floor plan:**
```
+---------------------------------------------+
|                   PALCO                      |
|                                              |
|  [Mesa Honra]                                |
|                                              |
|  [VIP-1] [VIP-2]    [VIP-3] [VIP-4]        |
|                                              |
|  [T-1]  [T-2]  [T-3]  [T-4]  [T-5]        |
|                                              |
|  [T-6]  [T-7]  [T-8]  [T-9]  [T-10]       |
|                                              |
|  === Corredor Servico ===                    |
|                                              |
|  [T-11] [T-12] [T-13] [T-14] [T-15]        |
|                                              |
|  [BAR]              [BUFFET/CAKE]  [SAIDA]  |
+---------------------------------------------+
```

**Sightline check:** Todas as mesas devem ter visao para o palco. Mesas atras de colunas ou pilares sao inaceitaveis para VIP.

### Step 5 — VIP & Protocol Seating

**Mesa de Honra (Head Table):**
- Posicao: centrada, frente ao palco ou ao centro da sala
- Composicao tipica: Anfitriao + Conjuge + Convidado de Honra + Conjuge + Orador principal + Key VIPs
- Alternancia homem-mulher (se protocolo formal)
- O anfitriao senta ao centro, convidado de honra a sua direita

**Ordem de proximidade ao palco/mesa de honra:**
1. VVIP (Chefes de Estado, C-suite do cliente principal)
2. VIP (Diretores, clientes-chave, speakers)
3. Convidados de Honra
4. Standard
5. Imprensa (mesa dedicada, com boa visibilidade mas sem interferir)

### Step 6 — Corporate Event Specifics

- **Separacao de concorrentes:** Minimo 2 mesas de distancia, nunca linha de visao directa
- **Client-host pairing:** Cada mesa de clientes tem 1-2 representantes do anfitriao
- **Speakers:** Mesa proxima do palco E da saida (para sairem discretamente para preparacao)
- **Sponsors:** Mesa conforme tier de patrocinio (Platinum perto do palco, Gold segunda fila)
- **Press table:** Boa visibilidade, acesso facil a saida, proximo de power outlets

### Step 7 — Social Event Specifics (Casamentos, Festas)

- **Mesas de familia:** Familia do noivo e da noiva em mesas separadas mas equidistantes da mesa principal
- **Mesa de solteiros:** EVITAR a "mesa dos solteiros" isolada — distribuir por mesas com casais
- **Mesa de criancas:** Se 5+ criancas, mesa dedicada com atividades; se menos, com pais
- **Sweetheart table:** Alternativa a mesa principal — casal sozinho, restantes em mesas redondas
- **Top table alternativas:** Meia-lua (visibilidade para todos), mesa redonda (mais inclusivo)

### Step 8 — Prepare for Last-Minute Changes

**Buffer seats:** 2-3 lugares vazios por cada 10 mesas (cadeiras disponiveis mas nao postas)

**Protocolo no-shows:**
1. Esperar 15min apos hora inicio
2. Remover lugar/cadeira ou redistribuir mesa
3. Reagrupar se mesa ficar com menos de 50% ocupacao
4. Nunca mover crachas/place cards a frente dos convidados

**Protocolo convidados inesperados:**
1. Verificar se esta na lista de contingencia
2. Se VIP: adicionar a mesa com buffer ou criar lugar na mesa de honra
3. Se standard: direcionar para mesa com lugares disponiveis
4. Informar catering imediatamente (extra plate)

**Rebalancing rules:**
- Mesa com menos de 50% ocupacao: fundir com outra
- Nunca mover convidados ja sentados (exceto pedido proprio)
- Ter 2-3 place cards em branco para emergencias

## Red Flags
- Sem verificacao de sightlines (mesas atras de colunas ou pilares)
- Espacamento insuficiente para cadeira de rodas (menos de 150cm)
- Mesa VIP longe do palco ou sem visibilidade
- Sem buffer seats para contingencias
- Plano de mesas finalizado tarde demais para impressao de place cards (min. 48h antes)
- Concorrentes na mesma mesa
- Mesa de solteiros isolada em eventos sociais
- Sem corredor de servico para catering (cadeiras costas-com-costas)
- Saida de emergencia bloqueada por mesas
- Sem protocolo definido para no-shows e convidados inesperados
- Place cards com fonte ilegivel (menos de 14pt)
- Floor plan sem escala (mesas nao cabem no espaco real)

## Portuguese Specifics
- **Mesa de Honra:** Tradicao forte em Portugal — esperada em galas, casamentos, e eventos institucionais
- **Protocolo estatal:** Ordem de precedencia portuguesa (ver atlas-protocol para detalhes completos)
- **Casamentos PT:** Tipicamente mesas redondas de 10; mesa principal com pais de ambos os noivos
- **Convencao de nomes:** Usar apelido completo nos place cards (dois apelidos)
- **Tratamento:** "Dr. Joao Silva Ferreira" nos place cards formais, nunca apenas "Joao"
- **Tendencia moderna:** Eventos corporativos PT cada vez mais usam table assignment (mesa sem lugar fixo) em vez de assigned seating

## 7. Status checklist per data point (Gate 7)

Every numeric value, name, or factual claim in **atlas-seating** output must carry an EXPLICIT label so the reader knows what to trust as-is and what needs verification before action.

- 🔵 **verified** — confirmed from a primary source (file, audit log, session memory, RAG hit, screen-shared confirmation)
- 🟡 **assumed** — plausible default that needs client/owner confirmation before being acted on
- 🟢 **projection** — estimate by design (timeline, cost, throughput, conversion); not verifiable today

**Why this gate exists:** outputs that mix verified facts with assumptions and projections without flagging them produce false certainty. The reader either over-trusts (acts on a guess) or under-trusts (ignores even the verified parts). Explicit labels restore signal.

**How to apply in atlas-seating:**

1. After drafting the deliverable, scan it for every concrete claim (number, name, date, metric, status, recommendation).
2. Attach one of the three labels inline; if you can't pick a label confidently, the claim isn't ready to ship.
3. Add a short citation in parentheses for 🔵 items (file path, source, dashboard) and a short condition for 🟡 / 🟢 items (what would confirm or refute it).
4. End the deliverable with a 1-line summary of how many items in each category, e.g. `Status mix: 8 🔵 · 3 🟡 · 2 🟢`.

❌ **NOT delivery-ready:**

```
Conversion rate is 18%. CAC is R$ 420. We will hit 1k MAU in Q3.
```

✅ **Delivery-ready:**

```
- Conversion rate: 18% 🔵 verified (Mixpanel funnel report 2026-05-19, n=1,242 sessions)
- CAC: R$ 420 🟡 assumed (calculated from May spend ÷ May customers; CFO has not signed off yet)
- 1k MAU in Q3 🟢 projection (linear extrapolation of last 8 weeks; assumes no churn spike)

Status mix: 1 🔵 · 1 🟡 · 1 🟢
```

**Ship checklist post-cliente-sync:**
- [ ] All 🟡 items confirmed (or downgraded to 🟢 / dropped)
- [ ] All 🔵 citations actually exist (no broken file paths, no imagined sources)
- [ ] All 🟢 projections labeled as such to the client — never presented as commitments
